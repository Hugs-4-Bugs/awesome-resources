JWT TOKEN STEPS ADDITIONAL INFORMATION:-


Step 1: Add the necessary dependencies to your project's pom.xml file
This step is like adding new tools to your project. You need to update a special file called pom.xml and include
 the dependencies you need for JWT token implementation.

Step 2: Create a class called "JwtAuthenticationEntryPoint" in the security package
Think of the security package as a special folder that holds important security-related classes. In this step, 
you're creating a class that will handle any unauthorized requests to your project. It's like having a person who 
says, "Hey, you can't come in without a valid pass!"

Step 3: Update the application.properties file with JwtSecurityKey and expiration time
The application.properties file contains important settings for your project. Here, you need to add two specific
settings: JwtSecurityKey (a secret key for generating the JWT tokens) and an expiration time (how long the token
is valid for). It's like writing down some important information for your project to use.

Step 4: Create a class called "JwtAuthenticationFilter" in the security package
This step is about creating a class that will check if users have a valid JWT token before allowing them to
access certain resources in your project. It's like having a security guard who checks if people have the 
correct pass before letting them enter certain areas.

Step 5: Create a class called "JwtTokenProvider" in the security package
In this step, you're creating a class that will generate the JWT token when a user logs in with their credentials.
This class will also verify the user's credentials and create the token. It's like having a person who creates a 
special pass for you after you prove who you are.

Step 6: Update the "sign-in" method in the authentication controller
This step involves making changes to the "sign-in" method in your authentication controller. Instead of just 
returning a success message, you need to modify it to return the JWT token after a successful login. It's like the
person who helps you get a special pass also giving you that pass.

Step 7: Create a class called "JwtAuthResponse" in the payload package
Whenever we signin and we are found to be valid user, this class will send a copy of the token to the postman for
sending back in the next transaction varification porpose. It will represent the response containing the JWT token.
It's like creating a special envelope to hold your pass and other important information.

Diffenrence between JwtAuthenticationEntryPoint JwtAuthenticationFilter.
JwtAuthenticationEntryPoint is responsible for handling unauthorized requests and sending appropriate error responses, while the
JwtAuthenticationFilter intercepts requests, performs the JWT authentication process, and sets the authenticated user in the 
SecurityContextHolder. They work together to ensure that only authenticated requests with valid JWT tokens can access protected resources.

JWT token brakedown:
Header: The header of a JWT token contains information about the type of token and the algorithm used for signing the token. 
	It is typically encoded as Base64Url.
Payload: The payload of a JWT token contains the actual data or claims that are being transmitted. Claims can include information 
	about the user, permissions, roles, or any other relevant data. Like the header, the payload is also Base64Url encoded.
Signature: The signature is a cryptographic signature created by combining the encoded header, encoded payload, and a secret key
	known only to the server. It is used to verify the integrity and authenticity of the token. The signature ensures that the 
	token has not been tampered with during transmission.
