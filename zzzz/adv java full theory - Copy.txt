What is ORM 
	ORM is a tool for parsing data between object to relation database table
	it is capable of handling various operations like inserting, updating, deleting etc
	It internally uses JDBC API to interact with the database.	

ORM working : we create a table (entity) in database and create entity class with same name as that of the table.
		  Entity class will have private variables whose name will match the DB table column names.
		  Now we create a object of entity class with the values to put in the table, ORM will map those values to the DB table.
		  By doing this we do not need to write any sql querries, if we need to add more data to the table, we create more new objects.

What is spring IOC?
	it is an area in spring boot framework which performs dependency injection and is responsible for bean-lifecycle
	Just like tomcat has Tomcat Container, which has logic to run servlets. Spring has SrprinIOC wich has logic to take care of objct creation when we use @Autowired.
	the object created this way are thread safe and secure

Dependency Injection- when @Autowired is used, the springIOC will automatically understand which objcet to create and inject in the address to the refference variable.
				Normally to create objects of interface we would use upacasting. But in springBoot we use this.

Bean LifeCycle:
	when we use @Autowired springIOC container creates a bean and then it performs dependency injection.
	After dependency injection is done, the very first method it runs is coustom init() method. Which runs only once.
	Then coustom utility() method will run any number of time.
	at last the coustom destroy() method will run and the bean life cycle will come to an end.
	complete life cycle is maintained by spring IOC.

	downloading dependencies is painfull in spring because for every frameworkd we have to download multiple dependency tags. To ovecome this in springBoot we use STARTER TAG.
